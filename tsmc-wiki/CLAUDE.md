## 2. 目錄結構

```
[project-root]/
├── CLAUDE.md               ← 本檔（Agent 記憶）
├── INDEX.md                ← 知識庫索引（每次新增檔案必須更新）
├── notes/                  ← 日常筆記、靈感、會議記錄
├── concepts/               ← 技術 / 領域概念深度整理
├── decisions/              ← 架構決策記錄（ADR 格式，唯讀）
├── daily/                  ← 每日情報（排程任務自動產生）
└── .claude/
    ├── commands/           ← /wiki-add、/wiki-search
    ├── skills/             ← note-writing
    ├── agents/             ← research-agent、writer-agent
    ├── hooks/              ← Session 結束自動記錄
    └── settings.json       ← Hook 設定
```

**各資料夾的差異**：

- `notes/` 是**流動的想法**，短、快、未經整理
- `concepts/` 是**沉澱後的知識**，結構完整、可當教材
- `decisions/` 是**不可撤回的決策**，ADR 格式，**唯讀**
- `daily/` 是**時效性情報**，由 Co-work 排程任務每日產生

## 3. 寫作規範

### 3.1 格式

每篇筆記開頭必須有 Frontmatter：

```yaml
---
title: "標題"
created_at: "YYYY-MM-DD"
updated_at: "YYYY-MM-DD"
tags: ["#tag1", "#tag2"]
status: "draft | reviewed | verified"
---
```

### 3.2 風格

- **繁體中文**；技術術語保留英文原文
- **第一人稱**，避免官腔
- **段落短**：每段 ≤ 3 行，條列優先
- **數據優先**：任何主張都要有可驗證的數字或來源
- **時間鎖定**：所有預測標註時間窗（例：「2025 Q4 預計量產」）

### 3.3 內部連結

使用 Markdown 相對連結：

```markdown
詳見 [技術護城河分析](./concepts/technology-moat.md)
```

## 6. Agent 行為準則

### 6.1 新增筆記時

1. 自動套用上述 Frontmatter
2. 標題使用 `# 標題` 開頭
3. 結尾必須回答：「這則筆記屬於哪個核心標籤？」
4. **更新 `INDEX.md`**（新增連結到對應分類）

### 6.2 搜尋筆記時

- 先查 `INDEX.md` 的分類索引
- 其次用 `Grep` 搜尋 tag
- 最後才全文搜尋

### 6.3 禁止事項

- ❌ 產生未標註來源的具體數字
- ❌ 修改 `decisions/` 既有 ADR（只能新增）
- ❌ 使用「絕對」、「一定」、「最好的」等無法驗證的形容詞
- ❌ 不更新 `INDEX.md` 就結束任務
